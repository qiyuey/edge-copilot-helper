# 更新版本号并创建标签

## 任务描述

当需要发布新版本时，执行以下步骤来更新版本号并创建 Git 标签：

## 操作步骤

1. **读取当前版本号**
   - 打开 `Cargo.toml` 文件
   - 找到 `version = "x.y.z"` 这一行
   - 记录当前版本号（例如：0.1.4）

2. **计算新版本号**
   - 将补丁版本号（第三位数字）加 1
   - 例如：0.1.4 → 0.1.5
   - 例如：0.2.3 → 0.2.4

3. **更新 Cargo.toml**
   - 将 `version = "旧版本号"` 修改为 `version = "新版本号"`
   - 保存文件

4. **提交版本号更新**
   - 执行：`git add Cargo.toml`
   - 执行：`git commit -m "更新版本号到 x.y.z"`（使用新版本号）

5. **创建 Git 标签**
   - 执行：`git tag -a vx.y.z -m "vx.y.z: 发布新版本"`（使用新版本号，例如：v0.1.5）

6. **推送到远程仓库**
   - 执行：`git push origin main`
   - 执行：`git push origin vx.y.z`（使用新标签名）

## 示例

假设当前版本是 0.1.4：

1. 新版本号：0.1.5
2. 更新 `Cargo.toml`：`version = "0.1.5"`
3. `git add Cargo.toml`
4. `git commit -m "更新版本号到 0.1.5"`
5. `git tag -a v0.1.5 -m "v0.1.5: 发布新版本"`
6. `git push origin main`
7. `git push origin v0.1.5`

## 注意事项

- 确保所有其他更改已提交（除了版本号更新）
- 确保已配置 Git 远程仓库
- 需要有推送权限
- 标签格式：`v` + 版本号（例如：v0.1.5）
